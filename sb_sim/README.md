# sb_sim

C++ host-side TileLink simulation library on top of [Switchboard](https://github.com/zeroasiccorp/switchboard). Provides message structs, agent classes, and a memory interface for driving TL-A/D channels from C++ test code against Verilator simulations.

## Directory structure

```
sb_sim/
├── include/
│   ├── tilelinklib.h    # TL enums, packed message structs, TLBundleParams
│   ├── tilelinklib.hpp  # TLAgent, ClientTLAgent, ManagerTLAgent
│   └── memifc.hpp       # ClientTLMemIfc — fesvr chunked_memif_t over TileLink
├── minimal/             # SB packet exchange (no TL) → [README](minimal/README.md)
├── tlloopback/          # TL loopback: client + manager → [README](tlloopback/README.md)
└── tlmem/               # TL memory + fesvr ELF load → [README](tlmem/README.md)
```

## C++ library

### `tilelinklib.h`

TL-A/D opcode enums (`TLAOpcode`, `TLDOpcode`), atomic op enums, packed wire-format structs (`TLMessageA`, `TLMessageD`), and `TLBundleParams`.

### `tilelinklib.hpp`

- **`TLAgent`** — base class: message builders (`put`, `putPartial`, `get`, `accessAck`, `accessAckData`), mask alignment. Call `set_TLBundleParams()` before use.
- **`ClientTLAgent`** — master side. `send_a()` / `recv_d()`. Connects to `<uri>_a.q` (tx) and `<uri>_d.q` (rx).
- **`ManagerTLAgent`** — slave side. `recv_a()` / `send_d()`.

### `memifc.hpp`

**`ClientTLMemIfc`** implements fesvr `chunked_memif_t` over `ClientTLAgent`. Splits bulk reads/writes into naturally-aligned power-of-2 TL transactions. Use with fesvr `load_elf`.

## Examples

| Example | Chisel DUT | Description |
|---------|-----------|-------------|
| [minimal](minimal/README.md) | `Minimal` | Raw 256-bit SB packet exchange (no TL) |

## CMake build targets

All examples share the same targets:

```sh
cmake -B build
cmake --build build --target verilator          # incremental build + run
cmake --build build --target verilator-rebuild  # force full recompile + run
cmake --build build --target clean-extra        # remove sim artifacts
```

FST waveform tracing:

```sh
cmake -B build -DTRACE=ON && cmake --build build --target verilator
```

## SV wrapper

Each example's `rtl_build/testbench.sv` is auto-generated by `build.py`. It uses macros from `switchboard.vh`:

- `QUEUE_TO_SB_SIM` — reads a `.q` file and drives DUT input wires via DPI
- `SB_TO_QUEUE_SIM` — captures DUT output wires and writes to a `.q` file

Queue URIs can be overridden at runtime via plusargs (e.g. `+io_client_0_a=<path>`).

See the [Switchboard documentation](https://github.com/zeroasiccorp/switchboard) for macro details.

## IDE setup (VS Code)

Each example has a `.vscode/settings.json.template`. Copy it and fill in local paths:

```sh
cp .vscode/settings.json.template .vscode/settings.json
```

| Field | Value |
|-------|-------|
| `python.defaultInterpreterPath` | Path to Python in your conda env, e.g. `~/miniconda3/envs/Switchboard/bin/python` |
| `python.analysis.extraPaths` | Switchboard repo root (dir containing the `switchboard/` package) |

`settings.json` is gitignored — changes stay local.
